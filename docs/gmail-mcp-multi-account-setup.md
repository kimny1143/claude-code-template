# Gmail MCP 複数アカウント設定手順書（C案: 転送+ラベル方式）

> conductor承認済み（2026-04-11）。kimny操作タイミングで実施。

## 概要

ビジネス用Gmailアカウントのメールをメインアカウント（kimny1143@gmail.com）に自動転送し、既存のClaude.ai Gmail MCPで両アカウントのメールを統合管理する。

**前提:**
- Claude.ai Gmail MCP: kimny1143@gmail.com に接続済み
- ビジネス用アカウント: `[ビジネス用メールアドレス]` ← kimnyが記入

---

## 手順

### Step 1: ビジネスGmailで転送設定（5分）

1. ビジネス用Gmailにログイン
2. **設定** → **メール転送とPOP/IMAP** タブ
3. 「転送先アドレスを追加」→ `kimny1143@gmail.com` を入力
4. kimny1143@gmail.com に確認メールが届く → リンクをクリックして承認
5. 「受信メールを kimny1143@gmail.com に転送」を選択
6. 「転送元にコピーを残す」を選択（推奨）
7. 保存

### Step 2: kimny1143@gmail.com でフィルタ設定（5分）

1. kimny1143@gmail.com にログイン
2. **設定** → **フィルタとブロック中のアドレス** タブ
3. 「新しいフィルタを作成」

**フィルタ条件:**
```
To: [ビジネス用メールアドレス]
```

**フィルタの操作:**
- [x] ラベルを付ける: `business`（新規作成）
- [x] 受信トレイをスキップ（アーカイブする）← 任意。メインの受信トレイを汚さない場合
- [ ] 既読にする ← オフ推奨
- [x] 一致するスレッドにもフィルタを適用する

4. フィルタを作成

### Step 3: 動作確認（5分）

1. ビジネスアカウントにテストメールを送る
2. kimny1143@gmail.com で受信確認
3. `business` ラベルが付与されていることを確認
4. Claude Code でテスト:

```
Gmail MCPで label:business のメールを検索して
```

→ `gmail_search_messages` で `query: "label:business"` が返ればOK

## 使い方（Claude Code セッション内）

### ビジネスメールの検索

```
ビジネスメールで「請求書」に関するメールを探して
→ gmail_search_messages(query: "label:business 請求書")
```

### ビジネスメールの読み取り

```
そのメールの内容を読んで
→ gmail_read_message(message_id: "...")
```

### 返信ドラフト作成

```
そのメールに返信のドラフトを作って
→ gmail_create_draft(...)
```

**注意:** ドラフトはkimny1143@gmail.com のドラフトフォルダに作成される。ビジネスアカウントからの送信が必要な場合は、ドラフト内容をコピーしてビジネスアカウントから手動送信すること。

## 制約事項

| 項目 | 状態 |
|------|------|
| ビジネスメールの読み取り | ✅ 可能（転送経由） |
| ビジネスメールの検索 | ✅ 可能（`label:business` で絞込） |
| ドラフト作成 | ✅ 可能（kimny1143名義） |
| ビジネスアカウントからの送信 | ❌ 手動操作が必要 |
| 転送遅延 | 数秒〜1分程度 |
| 転送元の操作（アーカイブ等） | ❌ 不可 |
