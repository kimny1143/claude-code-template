ひとことで

候補は2つ、役割が違う。まず frontend-design（公式・最安全）を社内で1件だけ試すのが、理解重視で一番わかりやすい。ui-design-brain は既存の道具と重なるので「出し比べ1回」で要否を判断。急ぎではありません。

frontend-designまず試す

見た目の方向づけ。「AIが作るとどれも同じ顔」を避け、その案件だけの意図的なデザインにさせる手引き。動くコードゼロ＝最安全。

ui-design-brain出し比べ後

部品の作法。ボタン・表・フォーム60種の正しい作り方。品質の下限は上がるが、既導入の道具と役割が重複。

戦略相談を予約
  • 主題を確定してから設計：「どの顧問か（音楽制作スタジオ向け）／読者／このページの唯一の仕事＝相談予約」を先に決める。左は主題不在で誰のLPにも見える。
  • 署名要素(signature)：主題の世界から1つ引く（右下＝ミキサーのフェーダー）。左は「大きな数字トリオ」＝典型テンプレ。
  • コピーは能動態で一貫：「戦略相談を予約」→押した後の通知も「予約しました」。左の "Get Started" は何が起きるか曖昧。
  • AI定番3クラスタを回避：紫→青グラデ／Inter既定／全部中央寄せ、を名指しで避ける。左はまさにそれ。

正直に：frontend-design は「良いデザインを保証」しません。没個性から遠ざける効き方です。案件情報が薄ければ効果も薄く、堅いB2Bで「冒険を1つ」が外すこともあります（＝トレードオフ）。

各スキルが「注入するもの」の実体

どちらも動く部分ゼロの手引き文書（読み込むとClaudeの作り方が変わるだけ）。ファイル全文を精読済み。

frontend-design

Anthropic 公式 · SKILL.md + LICENSE の2ファイル · 実行面ゼロ

  • プロセス規律：作る前に「色4-6色・書体・レイアウト・署名要素」の設計案を先出し → 自己批判（"誰にでも出す既定値か?"）→ 改訂 → 初めてコード化。
  • 美学の方向づけ：hero＝主張。大数字テンプレ・意味のない番号01/02/03・装飾モーションを戒める。
  • AI定番3クラスタを名指しで回避（クリーム+serif+テラコッタ／黒+アシッド/朱／新聞風hairline）。
  • コピー規律：能動態・動作名は全フロー一貫・エラーは謝らず具体的に。

ui-design-brain

community · MIT · 830★ · SKILL.md + components.md(1263行) · 実行面ゼロ

  • 60部品の作法カンペ：各部品に「別名・定義・best-practice・よくあるレイアウト」。例＝Button/Form/Table/Modal/Empty state…
  • 品質の下限：8pxグリッド・WCAG AA・44pxタッチ・フォーカス可視・hover/focus/active/disabled 全実装。
  • 禁止アンチパターン集：虹色バッジ・placeholderのみ・等weightボタン・予測画面へのspinner・"Click here"…
  • 5つのスタイルpreset（Modern SaaS / Apple風Minimal / Enterprise / Creative / Data Dashboard）。

正直な推奨

CCO の一本推奨

frontend-design を社内(template課)で1件だけ試験導入 → 実案件で before/after を体感 → 良ければ全課へ。ui-design-brain は「ui-ux-pro-max のみ vs +追加」で1回出し比べて、差が出てから採否。Taste／brand-guidelines／複製系は今回見送り。

過剰に推しません：frontend-design は良デザインを保証せず「没個性回避」の効き方。ui-design-brain は既存と重複するので、差が薄ければ入れない判断も正しい。

「今は何も足さない」も妥当：LPの本筋は Claude Design 製品の再走で、この導入はそれを止めないおまけの補強。最大手 ui-ux-pro-max は既にあります。

判断の選択肢

a 推奨どおり（frontend-design を社内お試し／ui-design-brain は出し比べ後／他は見送り）推奨でOK
b frontend-design を最初から全課へお試しを飛ばして即展開したい場合。
c 今は何も足さないClaude Design 製品の再走に集中する。
d もっと詳しく知りたい点がある気になる箇所を指定してください。
