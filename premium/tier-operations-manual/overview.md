# Tier制度 完全運用マニュアル

## はじめに

このマニュアルは、Claude Codeの複数インスタンス（課）がPRレビューを自律的に回すための**3段階レビュー制度**の完全ガイドです。

glasswerks AI Roomでは、10課体制で月間100+ PRを処理しています。全PRをCEOがレビューするのは非現実的。しかし、AIインスタンスに無制限のマージ権限を与えるのはリスクが高い。

この矛盾を解決するのがTier制度です。

---

## Tier制度の全体像

```
PR作成
  │
  ├─ 影響度チェック（判断フロー実行）
  │
  ├─ Tier 1: セルフレビュー → セルフマージ
  │   対象: docs、データファイル、コピーのみ、テスト追加のみ
  │   条件: CI全通過
  │
  ├─ Tier 2: ピアレビュー → ピア承認後マージ
  │   対象: コード変更あり（Tier 3に該当しないもの）
  │   条件: CI全通過 + 指定ピア課の承認
  │
  └─ Tier 3: コンダクターレビュー → CEO承認後マージ
      対象: セキュリティ/DB/価格/外部サービス/全課波及
      条件: CI全通過 + コンダクター承認 + CEO承認
```

### なぜ3段階なのか

- **Tier 1** は速度重視。ドキュメント修正で全員の手を止める必要はない
- **Tier 2** は品質保証。コード変更は別の視点が必要だが、全てをボトルネックにしない
- **Tier 3** は安全弁。影響範囲が大きい変更は慎重に

### 設計原則

1. **デフォルトはTier 3に上げる** — 判断に迷ったら安全側に倒す
2. **CI未通過はTier問わずマージ禁止** — 自動テストが最低限のゲートキーパー
3. **patrol報告に含める** — 全マージはコンダクターの巡回レポートで可視化される

## 導入ステップ（あなたのチームに適用するには）

### Step 1: 課の分類

あなたのチームの各インスタンス（課）をリストアップし、以下を決める:

| 決めること | 説明 |
|-----------|------|
| 課名 | 各インスタンスの役割名 |
| ワークスペース | 各課が担当するディレクトリ |
| Tier 2ペア | 相互レビューする相手課 |

**Tier 2ペアの決め方:**
- 同じ部署内の別課が最適（コンテキストが近い）
- 奇数課の場合は3課でローテーション
- 自分自身をTier 2相手にしない（意味がない）

### Step 2: CLAUDE.mdに判断フローを記載

各課のCLAUDE.mdに以下を追記:

```markdown
## PRレビュー権限

### 判断フロー（PR作成のたびに実行）

1. この変更は以下のいずれかに該当するか？
   - 価格・原価率に影響する
   - 認証・セキュリティに触る
   - DBスキーマを変更する
   - 本番環境設定を変える
   - 新しい外部サービスを導入する
   - [全課に波及する変更]
   → YES → Tier 3

2. コード変更を含むか？
   → NO → Tier 1
   → YES → Tier 2
```

### Step 3: Tier 2ペアテーブルを追記

```markdown
### Tier 2 レビュー担当

| 自分 | レビュー依頼先 |
|------|-------------|
| A課 | B課 |
| B課 | A課 |
| C課 | D課 |
| D課 | C課 |
```

### Step 4: PRタグルールを設定

| Tier | PRタグ | 例 |
|------|--------|-----|
| Tier 1 | `[self-review]` | `[self-review] docs: update README` |
| Tier 2 | `[peer-review: {課名}]` | `[peer-review: native課] feat: add button component` |
| Tier 3 | （タグなし、通常PR） | `feat: add Stripe webhook handler` |

### Step 5: patrol報告に含める

コンダクター（または管理インスタンス）の巡回レポートに以下を含める:

```
## マージ済みPR
| PR# | Tier | 課 | 概要 |
|-----|------|----|------|
| #42 | 1 | write課 | docs更新 |
| #43 | 2 | mued課 | ボタンコンポーネント追加 |
```
